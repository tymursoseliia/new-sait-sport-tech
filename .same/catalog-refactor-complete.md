- Адаптивная навигация

**Элементы управления:**

- Кнопка "Следующая" (disabled на последней странице)

**Логика:**
- Автоматический сброс на страницу 1 при изменении фильтров
- Автоматический сброс на страницу 1 при изменении сортировки
- Показ только текущей порции автомобилей

**Пример отображения:**
```
← Предыдущая  [1] 2 3 ... 10  Следующая →
```

## 💻 Технические детали:

### Компоненты:

**`CatalogClient.tsx`:**
- Клиентский компонент с состоянием фильтров
- `useState` для фильтров, сортировки, текущей страницы
- `useMemo` для оптимизации фильтрации и сортировки
- `useEffect` для сброса страницы при изменении фильтров

**`page.tsx`:**
- Server component
- `revalidate = 10` для near real-time обновлений
- Получение данных из Sanity через `getAllCars()`

### Стейт менеджмент:

```typescript
const [filters, setFilters] = useState({
  brand: '',
  year: '',
  priceFrom: '',
  priceTo: '',
  body: '',
  status: '',
});
const [sortBy, setSortBy] = useState('price-asc');
const [currentPage, setCurrentPage] = useState(1);
```

### Константы:

```typescript
const CARS_PER_PAGE = 9; // количество машин на страницу
```

## 🚀 Производительность:

### Оптимизации:
- `useMemo` для фильтрации и сортировки (пересчет только при изменении зависимостей)
- `useMemo` для пагинации (нарезка только текущей страницы)
- Данные из Sanity с `revalidate: 10` (кэш на 10 секунд)

### Скорость:
- Фильтрация: мгновенная (client-side)
- Переключение страниц: мгновенное (client-side)
- Загрузка данных: ~10ms (при попадании в кэш)

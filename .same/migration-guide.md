## 🔑 Шаг 1: Получите токен для записи в Sanity

### 1. Откройте Sanity Dashboard
Перейдите на https://www.sanity.io/manage

### 2. Войдите в аккаунт
Используйте GitHub или другой метод авторизации

### 3. Выберите проект
Найдите проект **ВОЛГА-АВТО** (Project ID: `9pu75wlq`)

### 4. Создайте API токен
1. Перейдите в **API** → **Tokens**
2. Нажмите **"Add API Token"**
3. Заполните:
   - **Name:** `Migration Token`
   - **Permissions:** выберите **"Editor"** (права на чтение и запись)
4. Нажмите **"Create"**
5. **ВАЖНО:** Скопируйте токен сразу! Он показывается только один раз

### 5. Добавьте токен в .env.local

Откройте файл `.env.local` и добавьте строку:

```env
SANITY_WRITE_TOKEN=ваш_скопированный_токен
```

Файл должен выглядеть так:

```env
# Sanity CMS
NEXT_PUBLIC_SANITY_PROJECT_ID=9pu75wlq
NEXT_PUBLIC_SANITY_DATASET=production
SANITY_WRITE_TOKEN=skRXxxx...ваш_токен

# Admin Panel
ADMIN_USERNAME=adminavto
ADMIN_PASSWORD=Волга-Авто1488
SESSION_SECRET=your-super-secret-key-change-this-in-production
```

## ⚠️ Важные замечания:

### Изображения
- URLs изображений сохраняются как текстовые поля (`imageUrl`)
- Для полной интеграции с Sanity CDN нужно будет вручную загрузить изображения в Sanity Studio
- После загрузки в Studio старые URL можно удалить

### Slug (ссылки)
- Все slug автоматически создаются из данных JSON
- Для автомобилей: `марка-модель-год` (например: `kia-k5-2021`)
- Для статей: используются существующие slug

### Статус автомобилей
- `available` → "В наличии"
- `order` → "Под заказ"
- `sold` → "Продано"

## 🛠️ Устранение неполадок:

### Ошибка: "SANITY_WRITE_TOKEN is not defined"
- Проверьте, что токен добавлен в `.env.local`
- Убедитесь, что нет пробелов или лишних символов
- Перезапустите терминал

### Ошибка: "Insufficient permissions"
- Токен должен иметь права **"Editor"**, а не "Viewer"
- Создайте новый токен с правильными правами

### Данные не появляются на сайте
- Обновите страницу (Ctrl + F5)
- Проверьте, что сервер dev запущен (`bun run dev`)
- Откройте консоль браузера на ошибки

### Дубликаты данных
- Скрипт использует `createOrReplace`, поэтому повторный запуск безопасен
- При повторном запуске существующие документы будут обновлены

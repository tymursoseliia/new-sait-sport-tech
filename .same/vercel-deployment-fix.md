## 📋 Что было сделано

### 1️⃣ Проблема

При деплое на Vercel возникала следующая ошибка:

```
⨯ Next.js can't recognize the exported `config` field in route "/page":
Unsupported node type "ConditionalExpression" at "dynamic".
Read More - https://nextjs.org/docs/messages/invalid-page-config
```

**Причина:**
Next.js 15 **не поддерживает условные выражения** (тернарные операторы) в экспортах конфигурации страниц (`dynamic`, `revalidate`).

### 2️⃣ Исправления

#### ✅ Файл: `src/app/page.tsx`

**Было (неправильно):**
```typescript
export const dynamic = process.env.NODE_ENV === 'production' ? 'auto' : 'force-dynamic';
export const revalidate = process.env.NODE_ENV === 'production' ? 60 : 0;
```

**Стало (правильно):**
```typescript
// Используем ISR с обновлением каждые 60 секунд
export const revalidate = 60;
```

#### ✅ Файл: `next.config.js`

**Было (неправильно):**
```javascript
swcMinify: true, // Использовать SWC минификатор
reactStrictMode: true, // Строгий режим React
```

**Стало (правильно):**
```javascript
reactStrictMode: true, // Строгий режим React
// swcMinify удалён - включён по умолчанию в Next.js 15
```

### 3️⃣ Отправлено на GitHub

## 📊 Технические детали

### ISR (Incremental Static Regeneration)

**Что это:**
ISR - это гибрид между статической генерацией (SSG) и серверным рендерингом (SSR).

**Как работает:**

1. При первом запросе страница генерируется статически
2. Страница кэшируется на 60 секунд
3. Следующие запросы получают закэшированную версию
4. Через 60 секунд страница регенерируется в фоне
5. Пользователи всегда получают быстрый ответ

**Преимущества:**

- ⚡ Быстрая загрузка (статические страницы)
- 🔄 Актуальные данные (обновление каждые 60 сек)
- 💰 Экономия ресурсов (меньше запросов к Sanity)
- 📈 Лучшее SEO (поисковики видят готовый контент)

### Next.js 15 изменения

В Next.js 15 появились новые ограничения:

❌ **Запрещено:**
```typescript
export const dynamic = process.env.NODE_ENV === 'production' ? 'auto' : 'force-dynamic';
```

✅ **Правильно:**
```typescript
export const revalidate = 60;
```

**Почему:**
Next.js компилятор должен знать значения во время сборки, а не во время выполнения.
